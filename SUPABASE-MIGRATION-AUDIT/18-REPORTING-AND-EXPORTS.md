# 18 — Reporting, Exports & Document Generation Audit

This document inventories every report, statement, ledger export, and print view in the WhiteBox GiftWorks application, mapping their visual content, schemas, logic, and role permissions for the future Supabase transition.

---

## 1. Gifting Spend Ledger Report

### Overview
*   **Trigger Elements:**
    *   `#btn-spend-ledger-csv` (CSV Export)
    *   `#btn-spend-ledger-print` (Print Ledger Statement)
*   **Location:** Gifting Dispatch Tab → "Spend Ledger" button → Modal popup (`#gifting-spend-ledger-modal`)
*   **Role-Based Access & Visibility Logic:**
    *   **Owner / Executive:** Pulls the **full organization ledger** (up to 142 mock entries in front-end seed).
    *   **Manager:** Pulls a **team-scoped ledger** (scrapes direct team transactions and aggregates).
    *   **Rep:** Pulls a **self-scoped ledger** (limited to gifts dispatched by the representative, up to 23 items).

---

### A. CSV Export Specification
When clicked, the script compiles transaction records into a comma-separated text blob:

*   **CSV File Name:** `Gifting_Spend_Ledger_YYYY-MM-DD.csv` (uses current ISO date).
*   **Header Row:** `Date,Recipient,Category,Confection,Amount,Rep,Team,Status`
*   **Fields Mapping (Front-end JS vs. Supabase Table):**

| Header Column | Front-end Object Key | Supabase Source Columns | Notes |
| :--- | :--- | :--- | :--- |
| **Date** | `p.timestamp` | `gifts.dispatched_at` | Text formatted as "May 24, 2026". |
| **Recipient** | `p.recipient` | `contacts.name` | Joined company or contact name. |
| **Category** | `p.category` | `gifts.category` | Lowercase value: `reach`, `retain`, `remember`, `reward`. |
| **Confection** | `p.confection` | `gifts.confection_type` | Box type representation (e.g. Sweet Box). |
| **Amount** | `p.amountText` | `gifts.amount` | Decimal value formatted with leading dollar sign. |
| **Rep** | `p.rep` | `profiles.name` | Joined name of AE who dispatched the gift. |
| **Team** | `p.team` | `profiles.workspace_id` | Mapped to user's branch or region. |
| **Status** | `p.status` | `gifts.status` | Order state: `dispatched`, `delivered`, `pending`. |

---

### B. Print Ledger Statement Specification
Invokes browser-native printing (`window.print()`).
*   **Print Layout Styling (CSS):**
    *   Targets media query `@media print` inside `style-v10-dashboard.css`.
    *   Hides the main navigation panel (`#sidebar-nav-container`), background overlays, action buttons (`.csv-btn`, `.print-btn`), search inputs, and page footers.
    *   Forces the ledger table container to expand to `width: 100%`, removing scrolls and scrollbars (`#printable-history-area::-webkit-scrollbar` etc.).
    *   Converts text colors to high-contrast dark gray on plain white backgrounds for ink-saving legibility.

## 2. Employee Performance & Touchpoint Trend Report

### Overview
*   **Trigger Elements:**
    *   `#btn-touchpoint-trend-csv` (CSV Export)
    *   `#btn-touchpoint-trend-print` (Print Performance Report)
*   **Location:** Employees Tab → Employee Profile click → Performance Trend Modal (`#employee-touchpoint-trend-modal`)
*   **Role-Based Access & Visibility Logic:**
    *   **Owner / Executive:** Can view and pull performance trend reports for **all employees**.
    *   **Manager:** Can view and pull reports only for **their direct team reports**.
    *   **Rep:** ❌ No access (Employees tab hidden).

---

### A. CSV Export Specification
Extracts historical client touchpoint logs generated by the target employee:

*   **CSV File Name:** `[Employee_Name]_Touchpoint_Trend_YYYY-MM-DD.csv` (e.g. `Tom_Collins_Touchpoint_Trend_2026-05-27.csv`).
*   **Header Row:** `Date,Type,Grade,Sector,Rep,Engagement Briefing & Notes`
*   **Fields Mapping (Front-end JS vs. Supabase Table):**

| Header Column | Front-end Object Key | Supabase Source Columns | Notes |
| :--- | :--- | :--- | :--- |
| **Date** | `p.date` | `activities.logged_at` | Format: "Feb 12". |
| **Type** | `p.type` | `activities.type` | Engagement type (e.g. Call, Email, Meeting). |
| **Grade** | `p.grade` | `activities.grade` | Quality index (A, B, C, D, F). |
| **Sector** | `p.sector` | `organizations.sector` | Mapped to client's industry sector. |
| **Rep** | `p.rep || empName` | `profiles.name` | Representative's full name. |
| **Engagement Notes** | `p.notes` | `activities.notes` | Text string containing logs. |

---

### B. Print Performance Report Specification
Triggers `window.print()` targeting employee profile details:
*   **Layout Formatting:** Prints a single-page document detailing the employee's name, performance index (A/B/C/D/F), average health index, active client counts, and chronological history table. Removes CSS backdrops and gradient widgets.

Fires a custom styled pop-up HTML generation engine before printing:

1.  Opens an isolated pop-up window: `window.open('', '', 'height=600,width=800')`.
2.  Generates a clean HTML structure referencing Google Fonts (`Outfit`):
    ```html
    <html>
      <head>
        <title>WhiteBox CRM - B2B Executive Activity Audit Statement</title>
        <link href="https://fonts.googleapis.com/css2?family=Outfit:wght@300;400;600;800&display=swap" rel="stylesheet">
        <style>
          body { font-family: 'Outfit', sans-serif; padding: 40px; }
          h1 { font-size: 24px; color: #7c3aed; }
          h2 { font-size: 11px; text-transform: uppercase; border-bottom: 2px solid #e2e8f0; padding-bottom: 12px; }
          table { width: 100%; border-collapse: collapse; }
          th { font-size: 11px; text-transform: uppercase; color: #475569; font-weight: 800; border-bottom: 2px solid #cbd5e1; }
          td { padding: 12px 10px; border-bottom: 1px solid #f1f5f9; font-size: 12.5px; }
        </style>
      </head>
      <body>
        <h1>WhiteBox CRM - B2B Executive Activity Statement</h1>
        <h2>CEO Gregory Sterling — Filter: [Timeframe] — Generated: [Timestamp]</h2>
        <table>
          <thead>
            <tr><th>Date</th><th>Activity Notes / Description</th></tr>
    ```

3.  Injects the current filtered ledger data (`#history-explorer-tbody` inner HTML).
4.  Fires browser print: `printWindow.print()`.
5.  Terminates the pop-up on completion: `printWindow.close()`.

## 4. Supabase Schema Migration Mapping

To support these reporting requirements on Supabase, the SQL queries must match the front-end expectations:

### A. Spend Ledger Query
To build the data array for the spend ledger CSV and print layout:
```sql
SELECT 
    g.dispatched_at AS timestamp,
    c.name AS recipient,
    g.category,
    g.confection_type AS confection,
    g.amount AS amountText,
    p.name AS rep,
    w.name AS team,
    g.status
FROM public.gifts g
JOIN public.contacts c ON c.id = g.contact_id
JOIN public.profiles p ON p.id = g.rep_id
JOIN public.workspaces w ON w.id = g.workspace_id
WHERE g.workspace_id = public.get_my_workspace()
ORDER BY g.dispatched_at DESC;
```

### B. Employee Touchpoint Query
To populate the Employee profile CSV/print:
```sql
SELECT 
    a.logged_at AS date,
    a.type,
    a.grade,
    o.sector,
    p.name AS rep,
    a.notes
FROM public.activities a
JOIN public.contacts c ON c.id = a.contact_id
JOIN public.organizations o ON o.id = c.org_id
JOIN public.profiles p ON p.id = a.rep_id
WHERE a.rep_id = $1
ORDER BY a.logged_at DESC;
```
